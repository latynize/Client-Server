stratum: 
Das Stratum ist eine Zahl, die die "Schicht" oder das Level des NTP-Servers in Bezug auf eine Referenzuhr 
(in der Regel eine Atomuhr) angibt. Ein Stratum-1-Server ist direkt mit der Referenzuhr verbunden, 
ein Stratum-2-Server ist mit einem Stratum-1-Server verbunden, und so weiter.

version: 
Dies ist die Version des NTP-Protokolls, das vom Server verwendet wird.

mode: 
Der Modus gibt an, wie der NTP-Server arbeitet. In Ihrem Fall ist der Modus 4, was bedeutet, 
dass der Server im Broadcast-Modus arbeitet.

poll: 
Das Polling-Intervall ist die Zeit in Sekunden zwischen aufeinanderfolgenden Anfragen an den NTP-Server.

precision: 
Die Genauigkeit des NTP-Servers. Dies ist ein Maß dafür, wie genau die Uhr des Servers ist. 
Die Angabe ist in einer 2er Potenz angegeben.

delay: 
Die Verzögerung ist die Zeit in Sekunden, die es dauert, bis ein Paket vom Client zum Server und zurück gelangt.

dispersion: 
Die Wurzel-Dispersion ist ein Maß für die maximale Fehlerwahrscheinlichkeit der Uhr des Servers. 
Es wird in Millisekunden angegeben.

2.	Vorteile gegenüber dem einfachen Daytime-Service:

•	Genauigkeit: 
	NTP kann die Zeit mit einer Genauigkeit von Millisekunden über das Internet und Mikrosekunden in LANs synchronisieren, während der Daytime-Service in der Regel nur eine Genauigkeit von einer Sekunde bietet.
•	Skalierbarkeit: 
	NTP kann eine große Anzahl von Clients bedienen, während der Daytime-Service eher für eine geringe Anzahl von Verbindungen ausgelegt ist.
•	Fehlerbehandlung: 
	NTP hat Mechanismen zur Erkennung und Korrektur von Fehlern, während der Daytime-Service dies nicht tut.

3.	Einsatz bei MacOS, Windows oder sonstiges:

•	MacOS: 
	MacOS verwendet NTP, um die Systemzeit zu synchronisieren. Sie können den NTP-Server, den MacOS verwendet, in den Systemeinstellungen ändern.
•	Windows: 
	Windows verwendet standardmäßig das Simple Network Time Protocol (SNTP), eine vereinfachte Version von NTP. Sie können jedoch auch NTP verwenden, indem Sie einen NTP-Server in den Internetzeiteinstellungen angeben.
•	Sonstiges: 
	Viele andere Systeme, einschließlich Linux-Distributionen und Netzwerkgeräte, verwenden ebenfalls NTP. Es ist auch möglich, einen eigenen NTP-Server aufzusetzen, wenn Sie eine genaue Zeitsynchronisation in Ihrem Netzwerk benötigen.
